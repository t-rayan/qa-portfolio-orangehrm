
# BUG_001 — Dashboard page loads after logout via browser Back button

**Module:** Login / Session Management
**Severity:** Medium
**Priority:** Medium
**Status:** Open
**Found by:** Narayan Thapa
**Date:** 24/03/2026

---

## Environment
- Browser: Chrome Version 146.0.7680.80 (Official Build) (arm64)
- OS: macOS Tahoe 26.3.1
- URL: https://opensource-demo.orangehrmlive.com
- Test Case: TC_009

---

## Steps to Reproduce
1. Navigate to https://opensource-demo.orangehrmlive.com
2. Login with Username: Admin / Password: admin123
3. Click the user icon at top right
4. Click Logout
5. Confirm you are back on the login page
6. Press the browser Back button

---

## Expected Result
Login page remains visible. The dashboard is not accessible
after logout. Session is fully terminated.

---

## Actual Result
The dashboard page loads and is visible after pressing the
Back button post-logout. However, clicking any links or
trying to access data redirects back to the login page.
The page visually appears but data and navigation are
inaccessible.

## Severity Justification
Rated Medium because:
- The dashboard UI loads which could confuse or alarm users
  into thinking they are still logged in
- No actual data is accessible — session is correctly
  terminated on the server side
- Could be escalated to High in apps that cache sensitive
  data on the page before session check completes

## Notes
This is a common session management issue where the browser
cache serves the previous page before the app can verify
the session has ended. Fix typically involves adding
Cache-Control: no-store headers or redirecting all
protected routes to login when no valid session exists.
